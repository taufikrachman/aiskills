# Perpajakan & Compliance Indonesia 🇮🇩

Implement Indonesian tax compliance in software: e-Faktur, PPh, SPT.

## Rules

### 1. Tax Types Overview
| Pajak | Rate | Applies To | Notes |
|-------|------|-----------|-------|
| PPh Final UMKM | 0.5% | UMKM revenue < Rp 4.8B/year | Per PP 23/2018 |
| PPh 21 | 5-30% | Employee salary (progressive) | Employer withholds |
| PPh 23 | 2%/15% | Services, royalties | Vendor pays |
| PPN (VAT) | 11% | Taxable goods & services | PKP only (≥ Rp 4.8B) |
| PPh 25 | Monthly | Corporate income tax installment | Based on previous year |

### 2. Tax Calculation in Transactions
```ts
interface Transaction {
  subtotal: number;
  ppnRate: number;        // 0.11 (11%) for PKP, 0 for non-PKP
  ppnAmount: number;       // subtotal * ppnRate
  pphFinalRate: number;    // 0.005 (0.5%) for UMKM Final
  pphFinalAmount: number;  // subtotal * pphFinalRate (deducted from seller)
  totalAmount: number;     // subtotal + ppnAmount
}
```
- PPN is ADDED to price (buyer pays).
- PPh Final is DEDUCTED from seller's revenue.

### 3. NPWP Validation
```ts
function validateNPWP(npwp: string): boolean {
  // Format: XX.XXX.XXX.X-XXX.XXX (15 digits)
  const clean = npwp.replace(/[.\-]/g, '');
  return /^\d{15}$/.test(clean);
}
```
- Validate format, not existence (DJP API needed for that).
- Store clean 15-digit string. Display formatted with dots.

### 4. e-Faktur (Electronic Tax Invoice)
- PKP must issue e-Faktur for every taxable transaction.
- Sequence number: auto-generated by DJP system.
- Required fields: NPWP seller, NPWP buyer, item details, PPN amount.
- Reference number: for credit notes and debit notes.
- Export to CSV for uploading to DJP e-Faktur desktop app.

### 5. Tax Report Export
```sql
-- Monthly PPN report
SELECT
  DATE_TRUNC('month', created_at) as bulan,
  COUNT(*) as jumlah_faktur,
  SUM(subtotal) as dpp,         -- Dasar Pengenaan Pajak
  SUM(ppn_amount) as ppn_keluaran
FROM transactions
WHERE ppn_amount > 0
GROUP BY DATE_TRUNC('month', created_at);

-- Monthly PPh Final report
SELECT
  DATE_TRUNC('month', created_at) as bulan,
  SUM(subtotal) as peredaran_bruto,
  SUM(pph_final_amount) as pph_final
FROM transactions
WHERE pph_final_amount > 0
GROUP BY DATE_TRUNC('month', created_at);
```
Export to CSV format accepted by DJP Online.

### 6. Common Pitfalls
- PPN threshold: only charge if annual revenue > Rp 4.8B (PKP).
- PPh Final: some sectors have different rates (construction 2-4%).
- Tax invoice numbering: must be sequential per year, NOT random.
- Credit note (faktur pajak pengganti): separate document, reference original.
- Regional differences: Batam FTZ has different PPN rules.

## Anti-Patterns
- ❌ Hardcoding tax rates (they change — use config)
- ❌ Rounding PPN incorrectly (always round up for tax)
- ❌ Mixing PPN and non-PPN transactions without flag
- ❌ Generating NPWP (let user input, just validate)
